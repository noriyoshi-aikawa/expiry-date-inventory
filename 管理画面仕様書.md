## 2. 各画面の仕様

### ① 商品登録画面（既存修正）

#### 追加セクション：「ロット・賞味期限管理」

数量情報セクションの直下に新規セクションを追加する。

**ロット管理トグル**

| 項目 | 仕様 |
|---|---|
| 表示名 | ロット管理 |
| 操作 | ON/OFF トグル |
| デフォルト | OFF |
| ON時の挙動 | 「出荷可能最低残日数」入力欄を表示。バリエーション在庫数がロット合計に自動反映され、直接編集不可になる |

**出荷可能最低残日数**

| 項目 | 仕様 |
|---|---|
| 表示名 | 出荷可能最低残日数 |
| 入力形式 | 数値（単位：日） |
| デフォルト | 未設定（制限なし） |
| 設定粒度 | 商品単位 |
| 動作 | 注文受付時、残日数がこの値を下回るロットは引き当てから自動除外 |
| 0または空欄 | 残日数による引き当て制限なし |

#### バリエーション設定への追加

- ロット管理ONのとき、バリエーション一覧の各行に「ロットを管理する →」ボタンを追加
- ロット管理ONのとき、バリエーション在庫数はロット合計が自動反映（直接編集不可）
- 合計在庫数の表示も「○個（ロット合計）」に更新

### ② ロット管理画面（新規作成）

**アクセス経路：** 商品登録画面 › バリエーション設定 › 「ロットを管理する」ボタン

#### ページヘッダー情報

- 商品名
- バリエーション名（SKUコード）
- 出荷可能最低残日数（商品設定値）
- ロット総在庫数（合計）

#### ロット追加フォーム

| 項目 | 必須/任意 | 形式 |
|---|---|---|
| ロットNo. | 必須 | 英数字、任意のコード |
| 賞味期限 | 必須 | 日付（YYYY-MM-DD） |
| 製造日 | 任意 | 日付（YYYY-MM-DD） |
| 入荷日 | 任意 | 日付（YYYY-MM-DD） |
| ロット在庫数 | 必須 | 整数 |

#### ロット一覧

| 列名 | 内容 |
|---|---|
| ロットNo. | 登録したロット番号 |
| 賞味期限 | YYYY-MM-DD形式 |
| 残日数 | 賞味期限 − 今日（自動計算） |
| 製造日 | 任意項目 |
| 入荷日 | 任意項目 |
| 在庫数 | 当該ロットの在庫数量 |
| 引き当てステータス | 「✓ 引き当て可能」または「⛔ 除外（残日数不足）」 |
| 操作 | 編集 / 削除 |

- ロットは賞味期限の近い順（FEFO）でソート表示
- 残日数が出荷可能最低残日数を下回るロットは赤色で「除外」表示
- 次回引き当て対象のロット（最も賞味期限が近く、引き当て可能なロット）をハイライト表示

#### CSV機能

- **CSVで一括登録：** ロット情報を一括アップロード
- **CSVでエクスポート：** 登録済みロット情報をダウンロード

### ⑤ 注文出力用フォーマットの設定（既存修正）

#### 追加項目

「ダウンロード可能な項目」リストに以下4項目を新規追加する。

| 項目名 | CSV出力値（単一ロット） | CSV出力値（複数ロット） |
|---|---|---|
| 引き当てロットNo. | `LOT-2025-001` | `LOT-2025-001\|LOT-2025-002` |
| 賞味期限（引き当てロット） | `2027-03-31` | `2026-06-19\|2026-08-27` |
| 注文時残日数 | `306` | `21\|90` |
| ロット別引き当て数量 | `2` | `2\|1` |

**複数ロットの場合：** パイプ（`|`）区切りで出力。4項目の順番は統一（ロットNo.の1番目 ↔ 賞味期限の1番目 ↔ 残日数の1番目 ↔ 数量の1番目）。

**ロット管理が有効でない商品の注文：** 4項目すべて空欄で出力。

## 3. 機能仕様

### FEFO自動引き当てロジック

注文確定時に以下の順序で処理を行う。

1. 対象バリエーションのロット一覧を取得
2. 賞味期限の近い順にソート（First Expired, First Out）
3. 残日数が**出荷可能最低残日数**以上のロットに絞り込む
4. 在庫が十分あれば、賞味期限が近いロットから順に引き当て
5. 在庫不足の場合は既存の在庫不足エラーフローに準拠
6. 複数ロットにまたがる場合はトランザクション管理で整合性を保証

### 賞味期限アラートバッチ処理

毎日深夜にバッチ処理を実行する。

1. 全ロットの残日数を計算（残日数 = 賞味期限 − 今日）
2. 残日数 ≦ アラート閾値（設定値）のロットを抽出
3. 対象ロットが1件以上あれば管理者へアラートメール送信
4. メール内容：商品名 / バリエーション / ロットNo. / 賞味期限 / 残日数 / 在庫数

### 残日数ゾーンの定義

| ゾーン | 条件 | 引き当て | アラート |
|---|---|---|---|
| 赤 | 残日数 < 出荷可能最低残日数 | 除外 | 送信 |
| オレンジ | 出荷可能最低残日数 ≦ 残日数 ≦ アラート閾値 | 対象 | 送信 |
| 緑 | 残日数 > アラート閾値 | 対象 | なし |

## 4. 設計上の決定事項

詳細は [設計判断ログ.md](設計判断ログ.md) を参照。

### 出荷可能最低残日数の設定粒度

- **MVPは商品単位で設定する**
- バリエーション単位の上書き設定はPhase 2候補
- Phase 2での実装方針：商品単位の値をデフォルト値とし、バリエーション設定画面で上書き可能にする

### 注文CSV複数ロット時の表示形式

- **パイプ区切り**（`LOT-001|LOT-002`形式）を採用
- 4項目の順番を統一することで倉庫担当がExcel上で対応関係を把握できる
